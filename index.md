## 6. AI Data Processing

### 6.1 How AI Features Work

When you use the Epoch Coach AI feature, the following data is sent to OpenAI's API for processing:

- Your query text (up to 80 characters)
- Contextual financial data necessary to generate a relevant reflection (e.g., budget status, financial path, goals, spending summary)

### 6.2 OpenAI Data Handling

OpenAI processes data on our behalf as a Data Processor. According to OpenAI's API data usage policy, data submitted through the API is not used to train or improve OpenAI's models.

### 6.3 Data Minimization

We apply data minimization principles to AI processing. Query text is truncated to 80 characters before storage. Only the minimum contextual data necessary to generate a meaningful response is transmitted to OpenAI.

### 6.4 AI Output Storage

AI-generated reflections and impact assessments are stored in your account data in our Convex database and are associated with your user ID.

> **Caution:** AI-generated content is not financial advice. The AI may produce inaccurate, incomplete, or inappropriate responses. Always exercise your own judgment regarding financial decisions.

## 9. Data Storage & Security

### 9.1 Storage Infrastructure

Server-side data is stored in Convex, a cloud database platform. Authentication data is managed by Clerk. Both services employ industry-standard security measures including encryption at rest and in transit (TLS 1.2+).

### 9.2 On-Device Storage

The following data is stored locally on your device only:

- Language preference (EN/TH) — stored in AsyncStorage; not synced to any server
- Authentication session tokens — stored in Expo Secure Store (encrypted, hardware-backed where available); cleared on logout

### 9.3 Security Measures

- All data in transit is encrypted using TLS 1.2 or higher
- Authentication is managed via Clerk with short-lived JWT session tokens
- On-device tokens are stored in hardware-backed secure storage (Expo Secure Store)
- Server-side access is authenticated and authorized on every API request
- We do not store passwords — authentication is fully delegated to Clerk
- Webhook communications are secured via SVIX with signature verification

### 9.4 Security Limitations

While we take reasonable measures to protect your data, no method of electronic transmission or storage is 100% secure. We cannot guarantee absolute security.

## 14. Cookies & Local Storage Technologies

Epoch Coach is a native mobile application and does not use browser cookies. However, the following equivalent technologies are used:

| Technology | Purpose |
|------------|---------|
| AsyncStorage | Language setting (EN/TH) only — not synced to server |
| Expo Secure Store | Clerk JWT authentication tokens — cleared on logout |
| Clerk Session Tokens | Short-lived JWTs for authenticating requests to Convex |
| Convex WebSocket | Persistent connection while app is open — no tracking beyond authenticated session |

None of these technologies are used for cross-app tracking, fingerprinting, or behavioral advertising targeting. The app does not use Google Analytics, Firebase Analytics, Mixpanel, Facebook SDK, or similar tracking tools.
